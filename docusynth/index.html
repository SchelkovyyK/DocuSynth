AST Parsing

Compiler-level extraction of function signatures, classes, parameters, and types

AST Parsing
Abstract Syntax Tree (AST) parsing is the foundation of DocuSynth's factual metadata extraction.

  • What Gets Extracted: Function signatures, names, parameters, return types, class definitions, imports, decorators.
  • Why AST Over Regex: Understands code structure semantically, handles nested/complex patterns, provides accurate locations.

LLM Synthesis

Deterministic documentation generation using grounded metadata

The LLM layer generates human-readable documentation from structured metadata.

  • Synthesis Steps: Module summarization, function documentation, cross-module coherence, markdown formatting.
  • Grounding Strategy: Only references extracted metadata, validates claims against AST data, never fabricates signatures.

Documentation Pipeline Workflow

  1. 1

    Repository Intake

    User selects or uploads a code repository. System initializes language-specific analyzers.

  2. 2

    Structural Code Analysis

    AST parser processes each file. Function signatures, classes, inheritance, decorators, imports are extracted.

  3. 3

    Metadata Validation

    Metadata is cleaned and normalized into a unified schema. Inconsistencies are flagged.

  4. 4

    Documentation Synthesis

    LLM produces summaries using structured metadata: purpose, parameters, returns, exceptions, examples.

  5. 5

    Diagram Construction

    Tool generates UML class diagrams, dependency graphs, and call graphs.

  6. 6

    Final Assembly

    System organizes all Markdown files, adds diagrams, and applies a consistent template.
